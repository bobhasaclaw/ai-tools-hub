# Research Low-Cost Stack

Prioritize value density.

## Selection rubric
Score each option:
1. Monthly cost (weight high)
2. Free tier quality/limits
3. Reliability/uptime
4. Setup complexity
5. Lock-in risk

## Research method
- Prefer official docs/pricing pages
- Note date checked
- Flag uncertain pricing as unverified
- Separate facts vs assumptions

## Output format
For each candidate:
- Cost now
- Cost at scale trigger
- Hard limits
- Best use case
- Why/when to avoid

Then provide:
- Best free option
- Best low-cost reliable option
- Migration path if user outgrows free tier

## Guardrails
- Avoid recommending paid tools unless clear ROI
- Prefer options with easy export/portability
- Include one fallback option per critical dependency
